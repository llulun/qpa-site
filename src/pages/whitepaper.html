概述

白皮书阐述 QPA 的架构设计、DOM 交互策略、调度与节流机制、反检测策略、日志与可视化体系、性能优化、测试方法与路线图。

架构设计

模块：Detection · Action · Scheduler · UI · Logger · Config · Guard。

数据流：检测列表 → 调度节奏 → 执行动作 → UI 展示 → Logger 记录 → Config 控制。

可插拔：模块解耦，参数化配置支持策略替换。

DOM 与事件模拟

选择器稳定性：优先稳定节点；避免过度依赖 class 名称。

点击事件：使用原生事件序列（Mouse/Pointer）。

状态读取：基于按钮文本/属性判断是否已点赞。

调度、节流与随机化

时间轮：以 refreshInterval 轮询，限制 maxLikesPerCycle。

节流：likeDelay / scrollInterval 控制最小间隔。

随机化：延时与步长加入微扰（±10–20%）。

反检测与风险控制

速率控制：避免短时大量点赞；分批完成。

行为多样性：滚动、暂停、延时与重试穿插。

过滤与阻止：filterKeywords 与 userBlockList 降低异常源。

日志与可视化

等级：INFO/WARN/ERROR；区分重要度。

内容：时间戳、事件名、参数摘要、结果状态。

持久化：面板列表上限 logMaxEntries；支持清理与导出。

性能优化

批处理：滚动后统一查询与处理。

缓存：已处理节点标识缓存。

可见性优先：优先处理可视区域。

测试与质量保证

跨浏览器：Chrome/Firefox 安装、运行与日志验证。

场景用例：弱网、长列表、过滤极端值、延时/步长边界。

回归检查：发布前复测检测→调度→点赞→日志主流程。

路线图

  • v2.8：更细粒度过滤规则与预设。
  • v2.9：策略插件化、个性化策略包。
  • v3.0：UI 重构、国际化（en-US）、更智能行为模拟。
